# AI相談ナビ 開発憲章 v1.0

> **正式名称**：AI相談ナビ 開発憲章 v1.0\
> **適用対象**：AI相談ナビ Prototype v1 以降

------------------------------------------------------------------------

# 1. 本憲章の目的

本憲章は、AI相談ナビ開発における最上位文書である。

本憲章は仕様書ではなく、プロジェクトの思想・判断基準・開発ルールを定義する。

------------------------------------------------------------------------

# 2. 基本原則

-   利用者を迷わせないことを最優先とする。
-   推測で仕様・思想を追加しない。
-   分からないことは必ず質問する。
-   設計確認前に実装しない。
-   品質を最優先する。

------------------------------------------------------------------------

# 3. プロジェクト目的

AI相談ナビは、AIへ相談する前に利用者の考えを整理し、AIへ正確に伝えられる状態を作ることを目的とする。

------------------------------------------------------------------------

# 4. 完成の定義

完成とは、実装完了ではない。

次の状態を目指す。

-   利用者が「AIって意外と使える」と感じること。
-   利用者が自然に「AI相談ナビを使ってみれば」と勧めたくなること。
-   利用者が迷わず利用できること。

------------------------------------------------------------------------

# 5. 正式版

-   React最新版を正式版とする。
-   最新ZIPを正式版とする。
-   HTML Prototype0は参考資料とする。
-   ReactとHTMLに差異がある場合はReactを優先する。

------------------------------------------------------------------------

# 6. AI開発ルール

AIは次を守る。

-   推測しない。
-   勝手に仕様変更しない。
-   判断理由を説明する。
-   不明点は質問する。
-   利用者第一を守る。

------------------------------------------------------------------------

# 7. 開発手順

1.  React解析
2.  実装状況整理
3.  確定事項整理
4.  確認事項整理
5.  設計
6.  承認
7.  実装
8.  レビュー
9.  納品

------------------------------------------------------------------------

# 8. 運用

プロジェクトには `docs` フォルダを設け、本憲章を保存する。

    docs/
    └── AI相談ナビ_開発憲章_v1.0.md

最新Reactプロジェクトには `docs` フォルダを含める。

------------------------------------------------------------------------

思想を変更する場合は十分な確認と合意を必要とする。

------------------------------------------------------------------------

# 開発チームの約束

私たちは、

-   利用者を迷わせない。
-   品質を妥協しない。
-   推測で仕様を決めない。
-   人が決めた思想をAIが守る。

以上をAI相談ナビ開発の基本原則とする。
